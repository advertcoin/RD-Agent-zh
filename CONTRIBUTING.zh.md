# 为 RD-Agent 贡献

我们欢迎对 RD-Agent 的贡献和建议。无论是解决问题、修复错误、增强文档，甚至是纠正拼写错误，每个贡献都是有价值的，有助于改进项目。

## 开始

要开始，您可以浏览问题列表或通过运行以下命令搜索代码库中的 `TODO:` 注释：
```sh
grep -r "TODO:"
```

## 如何贡献

1. **Fork 仓库**: 在 GitHub 上创建仓库的 fork。
2. **克隆仓库**: 将您的 fork 克隆到本地机器。
   ```sh
   git clone https://github.com/your-username/RD-Agent.git
   ```
3. **创建分支**: 为您的更改创建一个新分支。
   ```sh
   git checkout -b feature/your-feature-name
   ```
4. **进行更改**: 对代码库进行更改。
5. **提交更改**: 使用描述性的提交消息提交您的更改。
   ```sh
   git commit -m "Description of your changes"
   ```
6. **推送更改**: 将您的更改推送到您的 fork 仓库。
   ```sh
   git push origin feature/your-feature-name
   ```
7. **确保 CI 通过**: 确保您的代码通过 GitHub 上的自动 CI 检查。
8. **创建 Pull Request**: 从您的 fork 仓库向主仓库创建 pull request。

## 行为准则

请在与项目的所有互动中遵守 [行为准则](CODE_OF_CONDUCT.md)。

## 报告问题

如果您遇到任何问题或有改进建议，请在 GitHub 上打开一个 issue。

## 准则

- 确保您的代码遵循项目的编码标准。
- 编写清晰简洁的提交消息。
- 根据需要更新文档。
- 在提交 pull request 之前彻底测试您的更改。

感谢您为 RD-Agent 做出贡献！
